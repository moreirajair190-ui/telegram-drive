# TgPlayer v6.3 — dependências
#
# Instale com:
#     pip install -r requirements.txt
#
# Observação: use Python 3.10, 3.11 ou 3.12 (64 bits no Windows).
# (Python 3.13 pode dar problema com TgCrypto/Pyrogram — prefira 3.11/3.12.)

# Interface gráfica (Qt para Python).
# O pacote "PySide6" JÁ inclui PySide6-Essentials + PySide6-Addons
# (QtMultimedia, QtWebEngine, etc.). NÃO liste PySide6-Addons separado:
# isso causa conflito de versões no pip e quebra o build do .exe.
PySide6>=6.6,<6.9

# Observação: a aba "Acompanhamento" desenha os gráficos com QPainter puro
# (módulo charts.py). NÃO é necessário instalar PySide6-Charts/QtCharts.

# Cliente do Telegram (MTProto).
Pyrogram==2.0.106

# Acelera a criptografia do Pyrogram (recomendado).
# É OPCIONAL: o app funciona sem ele (apenas um pouco mais lento). Em alguns
# Pythons (ex.: 3.13) o TgCrypto não tem "wheel" pronto e tenta compilar em C,
# o que pode falhar. Se a instalação do TgCrypto falhar, o build_exe.bat
# continua mesmo assim e o app roda normalmente.
TgCrypto>=1.2.5 ; python_version < "3.13"

# Servidor HTTP local assíncrono usado para o streaming sob demanda.
aiohttp>=3.9

# (OPCIONAL) Backend de vídeo com libVLC EMBARCADO dentro da janela do app.
# Quando instalado (e com o VLC/libVLC presente no sistema), o player interno
# usa o motor do VLC — que inicia o streaming MUITO mais rápido — sem abrir o
# VLC externo. Se NÃO estiver instalado, o app cai automaticamente para o
# QMediaPlayer (codecs nativos do SO). Por isso esta linha pode ficar comentada.
#
#   pip install python-vlc
#
# python-vlc>=3.0.18

# Linter usado nos testes de fumaça do projeto (opcional em produção).
# pyflakes>=3.0

# Para gerar o executável (.exe) no Windows.
pyinstaller>=6.3
